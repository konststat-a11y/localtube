# VideoCloud

Локальний відеосайт на FastAPI. Застосунок запускається на комп'ютері користувача, зберігає відеофайли в локальній папці та показує приватні відео тільки тим користувачам, яким видано доступ.

## Можливості

- реєстрація, вхід і вихід через cookie-сесії;
- автоматичне створення першого адміністратора, якщо в базі ще немає адміністраторів;
- головна сторінка зі списком відео, сортуванням, категоріями та користувацькими розділами;
- перегляд відео через HTML5-плеєр і streaming endpoint з підтримкою HTTP `Range`;
- індивідуальний доступ до відео через таблицю `VideoAccess`;
- адмін-панель для користувачів, відео, сканування сховища та прав доступу;
- завантаження відео через сайт зі збереженням частинами до 10 ГБ;
- транскодування завантажених відео у браузерний MP4 через `ffmpeg`;
- генерація thumbnail через `ffmpeg`;
- профілі користувачів, аватари, історія переглядів, прогрес перегляду;
- лайки, дизлайки, список `Дивитися пізніше` та коментарі;
- видалення відео адміністратором або автором.

## Технологічний стек

- Python;
- FastAPI;
- Jinja2 templates;
- SQLAlchemy;
- Alembic;
- SQLite;
- passlib/bcrypt;
- Starlette `SessionMiddleware`;
- звичайні CSS і JavaScript без frontend-фреймворка.

## Структура проєкту

```text
app/
  main.py              # створення FastAPI app, middleware, routers, startup
  config.py            # шляхи, ліміти, розширення файлів, ffmpeg
  database.py          # SQLAlchemy engine/session/get_db
  models.py            # ORM-моделі
  auth.py              # авторизація, паролі, поточний користувач
  videos.py            # головна, перегляд, завантаження, streaming, реакції
  admin.py             # адмін-панель
  profile.py           # профіль і налаштування акаунта
  templates/           # Jinja2 templates
  static/              # CSS, JS, uploads avatars
migrations/
  env.py
  versions/            # Alembic migrations
video_storage/         # локальне сховище відео, не комітиться
run_server.bat         # запуск застосунку
run_migrations.bat     # застосування міграцій БД
requirements.txt
alembic.ini
product_spec.md
```

## Вимоги

- Windows;
- Python з підтримкою віртуального середовища;
- встановлений `ffmpeg`, якщо потрібні завантаження з транскодуванням і thumbnail.

`ffmpeg` шукається в такому порядку:

1. шлях із змінної середовища `VIDEOCLOUD_FFMPEG_PATH`;
2. локальний файл `tools/ffmpeg/bin/ffmpeg.exe`;
3. команда `ffmpeg` з `PATH`.

Без `ffmpeg` уже додані сумісні відео можуть відтворюватися, але транскодування та генерація прев'ю будуть недоступні.

## Перший запуск

Відкрийте PowerShell у корені проєкту.

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Застосуйте міграції бази даних:

```powershell
.\run_migrations.bat
```

Запустіть сервер:

```powershell
.\run_server.bat
```

Відкрийте сайт:

```text
http://127.0.0.1:8000/
```

## Міграції БД

Міграції не застосовуються автоматично під час старту застосунку. Перед першим запуском і після оновлень схеми використовуйте:

```powershell
.\run_migrations.bat
```

Якщо локальна SQLite-база вже містить таблиці, але ще не містить `alembic_version`, батник використовує сумісний runner: база позначається baseline-версією `001_baseline`, після чого оновлюється до `head`.

Батник також можна використовувати як wrapper над Alembic CLI:

```powershell
.\run_migrations.bat current
.\run_migrations.bat history
.\run_migrations.bat upgrade head
.\run_migrations.bat revision -m "describe change"
```

Під час зміни схеми БД потрібно:

- додати окрему міграцію в `migrations/versions/`;
- не видаляти користувацькі дані без окремого підтвердження;
- оновити `product_spec.md`, якщо змінюється продуктова логіка.

## Облікові записи

Під час старту застосунку створюється перший адміністратор, якщо в базі ще немає жодного користувача з `is_admin = true`.

Дані за замовчуванням для локальної розробки:

```text
login: admin
password: admin
```

Для реального використання в локальній мережі змініть пароль адміністратора та `SESSION_SECRET_KEY` в `app/config.py`.

## Робота з відео

Відео зберігаються в:

```text
video_storage/
```

Підтримувані формати для ручного додавання:

```text
.mp4, .webm, .mkv
```

Ручний сценарій:

1. покласти відеофайли в `video_storage/`;
2. увійти під адміністратором;
3. відкрити адмін-панель;
4. виконати сканування відео;
5. за потреби відредагувати метадані;
6. призначити користувачам доступ.

Сценарій завантаження через сайт:

1. авторизований користувач відкриває пункт завантаження;
2. сервер приймає файл частинами;
3. файл транскодується в `.mp4`;
4. створюється запис `Video`;
5. звичайним користувачам видаються записи `VideoAccess`;
6. користувач перенаправляється на сторінку перегляду.

Відео не віддаються через `StaticFiles`. Потік іде через:

```text
GET /videos/{video_id}/stream
```

Endpoint перевіряє авторизацію та доступ до конкретного відео до відкриття файлу.

## Адмін-панель

Адмін-панель доступна за адресою:

```text
http://127.0.0.1:8000/admin
```

Адміністратор може:

- створювати користувачів;
- створювати адміністраторів;
- сканувати `video_storage/`;
- редагувати назву, опис, автора та категорію відео;
- призначати індивідуальні доступи;
- видаляти відео.

## Типові проблеми

### Не запускається `run_server.bat`

Перевірте, що створено віртуальне середовище та встановлено залежності:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### Немає таблиць або помилка схеми БД

Запустіть міграції:

```powershell
.\run_migrations.bat
```

### Завантаження відео падає через ffmpeg

Перевірте, що `ffmpeg` доступний:

```powershell
ffmpeg -version
```

Або вкажіть шлях явно:

```powershell
$env:VIDEOCLOUD_FFMPEG_PATH = "C:\path\to\ffmpeg.exe"
```

### Відео не видаляється на Windows

Якщо файл зайнятий браузером або системою, застосунок приховує відео з інтерфейсу, перериває активні stream-відповіді та продовжує видалення у фоновій retry-задачі. Перед видаленням зі сторінки перегляду frontend зупиняє HTML5-плеєр.

## Безпека

Проєкт розрахований на локальне використання. Перед доступом із локальної мережі потрібно окремо перевірити:

- секрет сесії;
- пароль адміністратора;
- налаштування Windows Firewall;
- відсутність публічного доступу з інтернету;
- коректність прав доступу до відео.

Не відкривайте застосунок в інтернет без окремого опрацювання HTTPS, секретів, мережевої схеми та моделі загроз.
